EverythingTT Developer Guide

Architecture Overview

EverythingTT operates on a Distributed Monitoring model. The central Research Dashboard communicates with local C2 Agents (Python) to scan for desktop automation, while Remote Security Agents (UserScripts) report site-specific activity back to the dashboard via standard postMessage and fetch APIs.

Deployment Workflow

  1. Start the C2 Backend: python detector.py
  2. Install the Global Agent via GreasyFork.
  3. Monitor real-time interactions in the Live Monitored Sessions card.
  4. Simulate alerts via Cross-Site Broadcasts.

Contributing

Researchers can contribute by adding new detection modules. View our Technical Documentation for code samples and implementation details.
